## Overview

This repository implements Homework 5: Ontology-based Semantic Grounding. The goal is to model task objects from the AI Capstone baseline tasks and infer which objects are graspable by a robot gripper.

The submission uses a Python workflow managed by `uv`. RDF parsing and SPARQL queries are handled by RDFLib, and OWL/RDFS closure is handled by `owlrl`.

## Repository Contents

- `ontology/group-ontology.ttl`: Group 10 authored ontology.
- `ontology/imports/course-affordance.ttl`: Provided course ontology with a minimal Turtle syntax repair for `cap:hasApproxWidth`.
- `ontology/imports/course-alignment.ttl`: Provided SKOS alignment resource.

## Namespace Policy

The shared course namespace is:

```ttl
@prefix cap: <https://hcis.io/ontology/aicapstone/2026/> .
```

The Group 10 namespace is:

```ttl
@prefix g10: <https://hcis.io/ontology/aicapstone/2026/group10/> .
```

Course-level classes and properties are reused from `cap:`. Group-specific individuals, tasks, reusable affordance individuals, and role individuals are declared under `g10:`.

Task roles are represented as individuals under the Group 10 namespace. For example, `g10:targetObjectRole` is typed as `cap:TargetObject`, and object instances point to that individual with `cap:hasTaskRole`. This avoids using OWL classes such as `cap:TargetObject` directly as property values.

## Ontology Design

The ontology distinguishes four modeling layers:

| Layer | Example |
| --- | --- |
| Object type | `cap:Cup`, `cap:Knife`, `cap:ToyBlock`, `cap:Basket` |
| Task role | `cap:TargetObject`, `cap:ReferenceObject`, `cap:CollectableObject`, `cap:ContainerTarget` |
| Affordance | `cap:GraspingAffordance`, `cap:SupportAffordance`, `cap:ContainmentAffordance` |
| Instance | `g10:blueCup01`, `g10:knife01`, `g10:block01` |

The baseline objects are modeled as follows:

| Instance | Type | Role | Affordance |
| --- | --- | --- | --- |
| `g10:blueCup01` | `cap:Cup` | `cap:TargetObject` | grasping, stackability |
| `g10:pinkCup01` | `cap:Cup` | `cap:TargetObject` | grasping, stackability |
| `g10:knife01` | `cap:Knife` | `cap:TargetObject` | grasping |
| `g10:fork01` | `cap:Fork` | `cap:TargetObject` | grasping |
| `g10:plate01` | `cap:Plate` | `cap:ReferenceObject` | support |
| `g10:block01` | `cap:ToyBlock` | `cap:CollectableObject` | grasping |
| `g10:block02` | `cap:ToyBlock` | `cap:CollectableObject` | grasping |
| `g10:basket01` | `cap:Basket` | `cap:ContainerTarget` | containment |

Plate and basket are modeled as task-relevant objects, but they are not direct grasp targets in this baseline model.

## Key Axiom and Reasoning Pattern

The central class is `cap:GraspableObject`. The provided course resources already use this course-level term, and Group 10 adds the formal class definition in `ontology/group-ontology.ttl` because the starter affordance ontology does not include the full OWL axiom. It is defined as a physical object that has at least one grasping affordance:

```ttl
cap:GraspableObject
    a owl:Class ;
    rdfs:subClassOf cap:PhysicalObject ;
    owl:equivalentClass [
        a owl:Class ;
        owl:intersectionOf (
            cap:PhysicalObject
            [
                a owl:Restriction ;
                owl:onProperty cap:hasAffordance ;
                owl:someValuesFrom cap:GraspingAffordance
            ]
        )
    ] .
```

The source ontology does not manually assert Group 10 objects as `cap:GraspableObject`. The inferred graph is generated by `src/run_reasoning.py`, which:

1. Loads the course ontology and Group 10 ontology.
2. Loads the course SKOS alignment ontology for traceability.
3. Applies OWL/RDFS closure with `owlrl`.
4. Materializes the assignment-specific graspability rule for physical objects that have a `cap:GraspingAffordance`.
5. Writes `ontology/inferred-results.ttl`.
6. Runs the SPARQL queries and writes saved outputs under `results/`.

## Reproducibility

Run the workflow locally:

```bash
uv sync
uv run python src/run_reasoning.py
uv run pytest
```

The tests verify that Turtle files parse, graspable membership is not manually asserted for Group 10 instances, expected objects are inferred as graspable, and saved query output matches the expected result.

## Limitations

This submission intentionally stays within the baseline task scope. It does not model advanced gripper constraints, mass, deformability, task success criteria, or policy success/failure comparisons. The reasoning workflow is a compact educational implementation rather than a full OWL DL reasoner.
